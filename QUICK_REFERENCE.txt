WHAT WAS BROKEN?
  ✗ /api/tree returned incomplete file list (missing files from S3)
  ✗ Tree structure stored in S3 but never updated
  ✗ No user isolation - all users shared global tree
  ✗ Race conditions when multiple users edited simultaneously

WHAT'S FIXED?
  ✓ Tree now stored in PostgreSQL (source of truth)
  ✓ Auto-sync when notes created/deleted
  ✓ User isolation (each user has own tree)
  ✓ ACID transactions prevent race conditions
  ✓ Ready for PDF annotation feature

═══════════════════════════════════════════════════════════════════════════════

DATABASES CHANGES (3 NEW TABLES + 3 NEW COLUMNS)

NEW TABLES:
  1. app.tree_items (10,000 rows) - File hierarchy per user
  2. app.attachments (1,000 rows) - PDF/file tracking
  3. app.pdf_annotations (100 rows) - Drawing/markup data

NEW COLUMNS IN app.notes:
  1. s3_key TEXT - Reference to S3 binary file
  2. extracted_text TEXT - Text extracted from PDFs
  3. embedding vector(1536) - OpenAI embeddings for search

NEW EXTENSION:
  - pgvector - For semantic search capability

PERFORMANCE INDEXES:
  - 7 new indexes for fast tree lookups (~5-10MB overhead)

═══════════════════════════════════════════════════════════════════════════════

EXECUTION STEPS:

Step 1: Run SQL Migration
  Location: database/migrations/002_add_tree_and_vectors.sql
  Execute in PostgreSQL oghma database
  Time: <1 minute

Step 2: Deploy Code
  npm ci
  npm run build
  Deploy to production
  Time: ~5 minutes

Step 3: Rebuild Tree (API call)
  Method: POST /api/tree/rebuild
  Response: { orphanedNotesReattached: N }
  Purpose: Attach any existing orphaned notes to tree root
  Time: <1 minute

Step 4: Verify (API test)
  GET /api/tree - Should return all notes
  GET /api/notes - Should return all notes from PG
  POST /api/notes - Create test note, should auto-add to tree

NEW API ENDPOINTS:

GET  /api/pdf/annotations?noteId=123&attachmentId=456
  → Retrieve annotations for a PDF

POST /api/pdf/annotations
  Body: { noteId, attachmentId, annotationData }
  → Save/update PDF markup (for Konva.js drawing tool)

DELETE /api/pdf/annotations?id=123
  → Delete specific annotation

GET  /api/tree/rebuild
  → Check status (returns count of orphaned notes)

POST /api/tree/rebuild
  → Attach orphaned notes to tree root

═══════════════════════════════════════════════════════════════════════════════

MODIFIED ENDPOINTS (behavior change):

GET /api/tree
  Before: Queried S3 (slow, global tree)
  After: Queries PostgreSQL (fast, per-user)
  Auth: Required (new)

GET /api/notes
  Before: Queried S3
  After: Queries PostgreSQL
  Auth: Required (new)

POST /api/notes
  Before: Created note only
  After: Creates note + auto-adds to tree
  Benefit: Always in sync

DELETE /api/notes/[id]
  Before: Deleted note only
  After: Deletes note + removes from tree + deletes annotations
  Benefit: Cascading cleanup
