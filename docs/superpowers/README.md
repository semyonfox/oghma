# Historical Implementation Records

These docs are condensed records from earlier planning/build sessions. They are not current runbooks.

Use them for:

- understanding why a feature was added,
- finding the original affected areas,
- recovering acceptance criteria or verification ideas.

Do not use them as source-of-truth for current deployment, provider, or queue architecture. Prefer current code, migrations, [../DEPLOYMENT_GUIDE.md](../DEPLOYMENT_GUIDE.md), and [../../infra/HOMELAB.md](../../infra/HOMELAB.md).

## Layout

| Folder | Meaning |
|---|---|
| `specs/` | Product/design intent records |
| `plans/` | Condensed implementation plans |

The original verbose agent instructions, code snippets, and command transcripts were removed to avoid stale duplication. Important goals, files, and verification notes were retained.
