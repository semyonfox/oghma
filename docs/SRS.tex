\section{Introduction}
%==============================================================================

OghmaNotes is a web-based learning platform for students to create portable Markdown notes, upload PDFs, search intelligently, and ask AI questions with answers backed by their own materials.

\textit{For:} Development team, supervisors, maintainers.

%==============================================================================
\section{Scope}
%==============================================================================

\subsection{In Scope}

\begin{itemize}
    \item Web-based Markdown editor with offline support (PWA)
    \item PDF upload, text extraction, semantic chunking (500 tokens, 50--100 overlap)
    \item Multi-mode search (fuzzy keyword + semantic vector)
    \item RAG chat with citations to source materials
    \item User authentication (email/password, JWT tokens, password reset)
    \item Note organization (folder hierarchy)
    \item Quiz generation from notes/PDFs
    \item Spaced repetition flashcards (SM-2 algorithm)
    \item Canvas LMS integration for assignment sync
    \item Calendar/timetable view
    \item Analytics dashboard (study time, weak topics, progress)
    \item Full vault export
    \item Note tagging and bidirectional linking
    \item Responsive design (desktop, tablet)
\end{itemize}

\subsection{Out of Scope}

\begin{itemize}
    \item Real-time collaborative editing
    \item Native mobile apps (PWA only)
    \item Light mode (dark only)
    \item Instructor dashboards
    \item Video/audio transcription
\end{itemize}

\subsection{Goals}

\begin{enumerate}
    \item Deliver feature-complete product by end of semester
    \item Enable 3 people to work in parallel
    \item Auto-deploy from production branch
    \item Stay within budget constraints
\end{enumerate}

\section{Functional Requirements}
%==============================================================================

\subsection{Authentication \& Users}

\begin{itemize}
    \item Register with email/password
    \item Email verification required
    \item Login with JWT (24h expiry)
    \item Password reset via email (1h token, single-use)
    \item Logout and session management
    \item Passwords hashed with bcrypt (10+ rounds)
    \item Profile management (email, password, course info)
    \item Account deletion with 7-day soft-delete
\end{itemize}

\subsection{Notes}

\begin{itemize}
    \item Create, edit, delete notes in Markdown
    \item Live preview with formatting support
    \item Auto-save to IndexedDB every 5 seconds
    \item Organize into folder hierarchy
    \item Sync to S3 when online
    \item Conflict resolution (last-write-wins)
    \item Offline editing with auto-sync on reconnect
\end{itemize}

\subsection{PDFs \& Documents}

\begin{itemize}
    \item Upload PDFs up to 50MB (within 5GB quota)
    \item Extract text from PDFs
    \item Semantic chunking (500 tokens, 50--100 overlap)
    \item Generate and store embeddings (OpenAI, 1536 dims)
    \item Preserve page numbers and section metadata
    \item Display ingestion progress
    \item Handle errors gracefully (rate limits, API unavailable)
\end{itemize}

\subsection{Search}

\begin{itemize}
    \item Keyword search via PostgreSQL FULLTEXT (~5ms)
    \item Semantic search via pgvector embeddings (~50ms)
    \item Hybrid search combining both (~100ms)
    \item Filter by tags, module, document type, semester
    \item Display relevance scores and snippets
    \item Cmd+K overlay for keyboard-first search
\end{itemize}

\subsection{RAG Chat}

\begin{itemize}
    \item Chat interface for free-text questions
    \item Hybrid search to retrieve relevant chunks
    \item Send chunk text to LLM for answer generation
    \item Include citations (page number, section, chunk ID)
    \item Clickable citations linking to PDF location
    \item Stream LLM response token-by-token
    \item Rate responses (thumbs up/down)
    \item Handle LLM unavailability gracefully
\end{itemize}

\subsection{Quizzes}

\begin{itemize}
    \item Generate questions from notes/PDFs
    \item Select difficulty level (Easy/Medium/Hard)
    \item Generate 5--10 questions per session
    \item Support multiple-choice, short-answer, true/false
    \item Optional timed mode
    \item Display score and correct answers
    \item Track weak topics by performance
\end{itemize}

\subsection{Flashcards}

\begin{itemize}
    \item Create decks from notes or quiz questions
    \item SM-2 spaced repetition scheduling
    \item Daily review queue (``Due Today: N cards'')
    \item Flip cards and rate recall (1--5 scale)
    \item Update intervals based on ratings
    \item Persist state across devices
    \item Progress metrics (due, reviewed, mastery \%)
\end{itemize}

\subsection{Canvas LMS Integration}

\begin{itemize}
    \item OAuth connection to Canvas
    \item Import assignments with due dates
    \item Display in calendar view
    \item Auto-schedule study sessions before exams
    \item Auto-sync once per day
    \item On-demand sync button
    \item Handle sync errors gracefully
\end{itemize}

\subsection{Calendar \& Timetable}

\begin{itemize}
    \item Week and month views
    \item Display flashcard reviews, quizzes, LMS assignments
    \item Auto-schedule reviews before exams
    \item Click events to navigate to task
    \item Export to iCal format
\end{itemize}

\subsection{Analytics}

\begin{itemize}
    \item Study time tracking (weekly/monthly)
    \item Quiz scores over time
    \item Identify and rank weak topics
    \item Charts for performance and mastery progression
\end{itemize}

\subsection{Vault Management}

\begin{itemize}
    \item Export entire vault as ZIP archive
    \item Include all notes in Markdown with folder structure
    \item Include metadata (flashcards, quiz history, analytics) as JSON
    \item Embed timestamp and user ID in archive
\end{itemize}

\subsection{Tags \& Linking}

\begin{itemize}
    \item Add arbitrary tags to notes
    \item Create bidirectional links with wiki syntax (\texttt{[[Note Title]]})
    \item Display backlinks in note view
    \item Visualize knowledge graph of connections
    \item Filter notes by tags
\end{itemize}

\section{Non-Functional Requirements}
%==============================================================================

\subsection{Performance}

\begin{itemize}
    \item RAG query end-to-end: $\leq 3$ seconds
    \item PDF indexing (50MB): $\leq 2$ minutes
    \item Keyword search: $\leq 5$ ms; semantic: $\leq 50$ ms; hybrid: $\leq 100$ ms
    \item Quiz generation: $\leq 5$ seconds
    \item Note sync (500KB): $\leq 2$ seconds
    \item Cold page load: $\leq 4$ seconds; cached: $\leq 2$ seconds
    \item Backend handles 10+ concurrent requests
\end{itemize}

\subsection{Security}

\begin{itemize}
    \item APIs require JWT (except login, register, verify)
    \item Passwords hashed with bcrypt (salt $\geq 10$)
    \item HTTPS only (TLS 1.2+)
    \item CORS restricts approved origins
    \item Rate limiting on public endpoints
    \item Credentials in environment variables, encrypted at rest
    \item S3 presigned URLs expire in 1 hour
    \item Vector queries filtered by user ID
    \item Password reset tokens: 1 hour, single-use
    \item Email verification tokens: 24 hours
\end{itemize}

\subsection{Reliability}

\begin{itemize}
    \item 99\% availability during grading window
    \item Automated database backups (AWS RDS)
    \item Failed jobs retry 3 times with exponential backoff
    \item Graceful error handling for external API failures
    \item User sessions survive service restarts
    \item PWA reconnects within 30 seconds of network loss
\end{itemize}

\subsection{Usability}

\begin{itemize}
    \item Modern, clean design (Tailwind, Material Design 3 patterns)
    \item Register $\to$ upload PDF $\to$ ask question in $\leq 5$ minutes
    \item Real-time form validation
    \item Keyboard navigation support
    \item Responsive (desktop, tablet)
    \item Actionable error messages
    \item Dark mode only
\end{itemize}

\subsection{Maintainability}

\begin{itemize}
    \item ESLint and Prettier enforced
    \item OpenAPI 3.1 API documentation
    \item Version-controlled database migrations
    \item Structured JSON logging
    \item README with setup and deployment
    \item Automated dependency scanning (Dependabot)
\end{itemize}

\section{Database}
%==============================================================================

\subsection{ID Strategy}

All ID columns use UUID v7:
\begin{itemize}
    \item Cryptographically secure
    \item Sortable by timestamp
    \item Generated server-side
    \item Applied to: users, notes, documents, chunks
\end{itemize}

\subsection{Core Tables}

\begin{itemize}
    \item \texttt{app.login} --- User accounts, passwords (bcrypt)
    \item \texttt{app.notes} --- Note content, metadata
    \item \texttt{app.documents} --- PDF metadata and S3 paths
    \item \texttt{app.chunks} --- Text chunks with embeddings and metadata
\end{itemize}

\subsection{Search Columns}

Notes table includes:
\begin{itemize}
    \item \texttt{extracted\_text} --- Full text for FULLTEXT search
    \item \texttt{embedding} --- Vector (1536-dim) for semantic search
    \item \texttt{search\_vector} --- PostgreSQL tsvector for indexing
    \item \texttt{relative\_path} --- For organizing results
\end{itemize}

\subsection{Indexes}

\begin{itemize}
    \item GIN index on \texttt{search\_vector} (FULLTEXT)
    \item ivfflat index on \texttt{embedding} (semantic)
    \item Compound indexes on \texttt{user\_id + created\_at}, \texttt{user\_id + updated\_at}
\end{itemize}

\section*{Appendix: Glossary}
%==============================================================================

\begin{itemize}
    \item \textbf{RAG}: Retrieval-Augmented Generation. Use vectors to retrieve relevant text; send text to LLM for grounded answers with citations.
    \item \textbf{PWA}: Progressive Web App. Installable web app with offline support.
    \item \textbf{SM-2}: SuperMemo 2 spaced repetition algorithm.
    \item \textbf{JWT}: JSON Web Token for session authentication.
    \item \textbf{Vector Embedding}: Numerical representation (1536-dim) for semantic similarity search.
    \item \textbf{Chunking}: Splitting documents into segments (500 tokens, 50--100 overlap) for embedding and retrieval.
    \item \textbf{pgvector}: PostgreSQL extension for vector similarity search (HNSW indexing).
    \item \textbf{FULLTEXT}: PostgreSQL full-text search for keyword matching.
    \item \textbf{Bidirectional Links}: Wiki-style note connections where both notes display the relationship.
\end{itemize}
