\section{What This Is}
%==============================================================================

OghmaNotes is a web-based learning platform for students to create Markdown notes, upload PDFs, search intelligently, and ask AI questions with answers backed by their own materials.

This SRS describes what we're building and how. It's a living document---the plan stays, the implementation adapts.

\textit{For:} Development team, supervisors, maintainers.

%==============================================================================
\section{Scope}
%==============================================================================

\subsection{In Scope (MVP)}

\begin{itemize}
    \item Web-based Markdown editor with offline support (PWA)
    \item PDF upload, annotation, text extraction
    \item Multi-mode search (fuzzy keyword + semantic/vector)
    \item RAG chat: ask questions, get answers with citations
    \item User authentication (email/password)
    \item Note organization (folder hierarchy)
    \item Responsive design (desktop, tablet)
\end{itemize}

\subsection{Out of Scope (MVP)}

\begin{itemize}
    \item Light mode (dark only)
    \item Real-time collaborative editing
    \item Native mobile apps (PWA sufficient)
    \item Instructor dashboards
    \item Video/audio transcription
\end{itemize}

\subsection{Goals}

\begin{enumerate}
    \item Finish MVP in 6 weeks (semester deadline)
    \item Enable 3 people to work in parallel
    \item Auto-deploy from production branch
    \item Stay within AWS free tier + modest budget
\end{enumerate}

\section{Why We Changed Our Tech Stack}
%==============================================================================

The original plan specified MariaDB and microservices. We adapted based on team expertise and MVP pragmatism. This is not a failure to plan---it's responding to reality.

\subsection*{Database: MariaDB $\rightarrow$ PostgreSQL + pgvector}

\textbf{Original:} MariaDB with native vector columns.

\textbf{Changed to:} PostgreSQL 12+ with pgvector extension.

\textbf{Why:} Team knows PostgreSQL. pgvector is well-maintained and widely used. Both handle vector search equally well. No functional loss.

\subsection*{Architecture: Microservices $\rightarrow$ Next.js Monolith}

\textbf{Original:} 3 services (frontend, API, RAG pipeline) + background workers.

\textbf{Changed to:} Single Next.js app (frontend + API routes).

\textbf{Why:} Faster iteration, cleaner deployment. MVP doesn't need independent scaling. Can refactor later if needed.

\subsection*{Background Jobs: Redis + BullMQ $\rightarrow$ Deferred}

\textbf{Original:} Redis + BullMQ for async PDF processing.

\textbf{Changed to:} Synchronous for now, async later if needed.

\textbf{Why:} Can start with sync processing. If large PDFs slow down the system, add async in Phase 2.

\subsection*{LLM Key Management: Backend-Managed $\rightarrow$ User-Managed}

\textbf{Original:} Backend stores and manages API keys.

\textbf{Changed to:} Users provide own keys (stored in browser, never on backend).

\textbf{Why:} Simpler for us. Users pay their provider directly (transparent costs). Supports any LLM (OpenAI, Anthropic, local). Better privacy.

\section{What We're Building}
%==============================================================================

\subsection{Authentication \& Users}

\begin{itemize}
    \item Register with email/password
    \item Email verification
    \item Login with JWT tokens (24h expiry)
    \item Password reset with email link (1h token)
    \item Logout and session management
    \item Bcrypt hashing (10+ rounds)
\end{itemize}

\subsection{Notes}

\begin{itemize}
    \item Create, edit, delete notes in Markdown
    \item Auto-save to IndexedDB (every 5s)
    \item Folder hierarchy organization
    \item Sync to S3 when online
    \item View note history/versions (future)
    \item Bidirectional links (future)
    \item Tags (future)
\end{itemize}

\subsection{PDFs \& Documents}

\begin{itemize}
    \item Upload PDFs up to 50MB
    \item View and annotate (highlighting, notes)
    \item Text extraction for search
    \item Semantic chunking (500 tokens, 50-100 overlap)
    \item Generate embeddings (OpenAI or user's provider)
    \item OCR for scanned PDFs (future)
\end{itemize}

\subsection{Search (Phase 1)}

\begin{itemize}
    \item Fuzzy search: keyword matching with typo tolerance
    \item Semantic search: meaning-based results via vectors
    \item Search by filename and content
    \item Relevance scoring for both types
    \item Cmd+K overlay UI (keyboard-first)
    \item Sort results by relevance or date
    \item Pagination (20 results per page)
    \item Date range filtering (future)
\end{itemize}

\textit{Full technical details in SEARCH\_ARCHITECTURE\_PLAN.md}

\subsection{RAG Chat with Citations (Phase 2)}

\begin{itemize}
    \item Chat interface for questions
    \item Retrieve relevant document chunks via semantic search
    \item Send context to LLM with user's API key
    \item Generate answers with citations (source + page/section)
    \item Stream responses token-by-token
    \item Conversation history storage
    \item Feedback/rating on answers (future)
\end{itemize}

\subsection{Quiz Generation (Phase 3)}

\begin{itemize}
    \item Generate questions from notes/PDFs
    \item Multiple-choice, short-answer types
    \item Configurable difficulty levels
    \item Timed mode option
    \item Score tracking
\end{itemize}

\subsection{Spaced Repetition Flashcards (Phase 3)}

\begin{itemize}
    \item Create decks from notes or quiz questions
    \item SM-2 algorithm for scheduling reviews
    \item Daily review queue
    \item Mastery tracking (1-5 scale)
    \item Persist state across devices
    \item Statistics (cards mastered, retention rates)
\end{itemize}

\subsection{Offline Support}

\begin{itemize}
    \item Installable as PWA
    \item Edit notes offline (sync when back online)
    \item Queue changes locally
    \item Auto-sync on reconnect (<30s)
    \item Conflict resolution (last-write-wins)
\end{itemize}

\subsection{LMS Integration (Phase 3)}

\begin{itemize}
    \item OAuth 2.0 Canvas connection
    \item Import assignments and due dates
    \item Display in calendar view
    \item Export quiz results to Canvas gradebook
    \item Manual sync + refresh option
\end{itemize}

\subsection{Calendar \& Timetable (Phase 3)}

\begin{itemize}
    \item Calendar view of note creation/modification dates
    \item Timetable showing class schedules
    \item Highlight assignment due dates
    \item Color-code by course
    \item Week/month/year views
\end{itemize}

\subsection{Analytics Dashboard (Phase 3)}

\begin{itemize}
    \item Study time tracking (per subject)

\section{Non-Functional Requirements}

\subsection{Performance Targets}

\begin{itemize}
    \item Page cold load: $\leq 4$s
    \item Page cached load: $\leq 2$s
    \item Note sync (500KB): $\leq 2$s
    \item Keyword search: $\leq 5$ms
    \item Semantic search: $\leq 50$ms
    \item Hybrid search: $\leq 100$ms
    \item PDF indexing (50MB): $\leq 2$ min
    \item RAG end-to-end response: $\leq 3$s
\end{itemize}

\subsection{Security}

\begin{itemize}
    \item Passwords hashed with bcrypt (10+ rounds)
    \item HTTPS only (TLS 1.2+)
    \item JWT tokens in httpOnly cookies
    \item CORS policy enforced
    \item User data isolated (users can only see own notes)
    \item API keys in environment variables only
    \item S3 presigned URLs with 1h expiry
    \item Password reset tokens: 1h, single-use
    \item Email verification tokens: 24h
    \item Rate limiting (future)
\end{itemize}

\subsection{Reliability}

\begin{itemize}
    \item 99\% availability (AWS Amplify SLA)
    \item Automated database backups (RDS)
    \item Session persistence across restarts
    \item PWA offline-first design
    \item Graceful error handling
\end{itemize}

\subsection{Usability}

\begin{itemize}
    \item Modern design (Tailwind CSS 4)
    \item Dark mode
    \item Keyboard shortcuts (Cmd+K, etc.)
    \item Responsive layout (desktop, tablet, mobile)
    \item Clear error messages
    \item <5 min learning curve for core features
    \item WCAG AA accessibility (future)
\end{itemize}

\subsection{Maintainability}

\begin{itemize}
    \item ESLint + Prettier enforced
    \item SQL migrations tracked

    \item Structured logging (AWS CloudWatch)
    \item 70%+ test coverage for core logic
\end{itemize}

\section{Database Schema}
%==============================================================================

\subsection{Core Tables}

\begin{itemize}
    \item \texttt{app.login} --- User accounts, hashed passwords
    \item \texttt{app.notes} --- Note content, metadata, search/embedding columns
    \item \texttt{app.tree\_items} --- Folder hierarchy
    \item \texttt{app.attachments} --- Uploaded PDFs and files
    \item \texttt{app.pdf\_annotations} --- Highlights, notes on PDFs
    \item \texttt{app.quiz\_questions} --- Quiz questions (Phase 3)
    \item \texttt{app.flashcards} --- Flashcard decks (Phase 3)
\end{itemize}

\subsection{Search Columns (Phase 1)}

\texttt{app.notes} will include:
\begin{itemize}
    \item \texttt{extracted\_text} --- Full text from PDFs
    \item \texttt{embedding} --- Vector (1536-dim) for semantic search
    \item \texttt{search\_vector} --- PostgreSQL tsvector for full-text search
    \item \texttt{relative\_path} --- For organizing search results
\end{itemize}

\subsection{Indexes}

\begin{itemize}
    \item GIN index on \texttt{search\_vector} (full-text)
    \item ivfflat index on \texttt{embedding} (semantic)
    \item Compound indexes on \texttt{user\_id + created\_at} and \texttt{user\_id + updated\_at}
\end{itemize}

\section{LLM Strategy: User-Managed Keys}
%==============================================================================

Users bring their own API keys. We never see them, don't store them, don't charge for them.

\begin{itemize}
    \item User provides key (OpenAI, Anthropic, etc.)
    \item Key stored in browser (never sent to backend)
    \item Frontend calls LLM directly
    \item Backend coordinates search only
    \item User pays provider directly
    \item Team tests with free local Qwen model
\end{itemize}
