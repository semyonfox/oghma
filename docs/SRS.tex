\section{Introduction}
%==============================================================================

\subsection{What is This?}

OghmaNotes is a web app for students. You write notes, upload PDFs, search them with AI, and ask questions to get answers backed by your own materials.

This document describes what we're building and how. It changes as we learn things. If something here doesn't make sense or changes, we update it.

\textit{For:} The dev team, supervisors, anyone maintaining this later.

\subsection{Status}

\textbf{Version 2.1} (March 6, 2025) --- Living document. Core + search in Phase 1, RAG coming Phase 2.

\subsection{What This Is}

OghmaNotes is a web-based learning platform for students to organize notes, upload PDFs, search intelligently, and get AI-powered answers backed by their own materials.

\textbf{Working now:}
\begin{itemize}
    \item Create and manage Markdown notes with offline support
    \item Upload and annotate PDFs
\end{itemize}

\textbf{Building soon:}
\begin{itemize}
    \item Ask questions and get answers with citations from your notes
    \item Smart search (fuzzy + semantic)
    \item Generate quizzes and flashcards
    \item Track learning progress
    \item Connect to Canvas LMS
\end{itemize}

\subsubsection{By Semester End (MVP)}
\begin{itemize}
    \item Notes: create, edit, organize in folders, sync offline
    \item PDFs: upload, annotate, extract text
    \item Search: fast fuzzy + semantic with keyboard shortcut
    \item RAG: ask questions, get answers with citations
    \item Auth: email/password login
    \item Deploy: works on desktop, tablet, browsers
\end{itemize}

\subsubsection{Not Included}
\begin{itemize}
    \item Light mode (dark only for now)
    \item Real-time collab editing
    \item Native mobile apps (web is enough)
    \item Instructor dashboards
    \item Video/audio transcription
\end{itemize}

\subsubsection{Goals}
\begin{enumerate}
    \item Finish MVP in 6 weeks (semester deadline)
    \item 3 people working in parallel without stepping on each other
    \item Auto-deploy from \texttt{prod} branch
    \item Stay within budget (AWS free tier + modest spend)
\end{enumerate}

\section{What We're Using}

Pragmatic choices based on team expertise and what actually works.

\subsection{Frontend}
\begin{itemize}
    \item \textbf{Framework:} Next.js 16 (App Router, SSR/SSG)
    \item \textbf{Language:} TypeScript + JSX (type safety)
    \item \textbf{UI:} React 19 + Tailwind CSS 4
    \item \textbf{Editor:} Lexical (rich text, better UX than Markdown-only)
    \item \textbf{PDF Viewer:} PDF.js + react-pdf (client-side)
    \item \textbf{State:} Zustand (lightweight)
    \item \textbf{Theme:} Dark mode only (MVP)
\end{itemize}

\subsection{Backend (Next.js API Routes)}
\begin{itemize}
    \item \textbf{Framework:} Next.js 16 (no separate backend service)
    \item \textbf{Auth:} bcryptjs (hashing) + JWT (tokens)
    \item \textbf{Email:} Nodemailer + AWS SES
    \item \textbf{Database:} PostgreSQL 12+ (postgres.js driver)
    \item \textbf{Storage:} AWS S3 (file uploads, PDFs, exports)
    \item \textbf{Middleware:} Custom CORS, session validation
\end{itemize}

\subsection{Database \& Search}
\begin{itemize}
    \item \textbf{Primary:} PostgreSQL 12+ (RDS or local)
    \item \textbf{Vector Search:} pgvector extension (1536-dim embeddings)
    \item \textbf{Full-Text Search:} PostgreSQL FULLTEXT (not yet implemented)
    \item \textbf{Migrations:} Manual SQL files (in \texttt{database/} folder)
\end{itemize}

\subsection{LLM \& AI (User-Managed Keys)}

\textbf{Key Innovation:} OghmaNotes does \textit{not} store LLM API keys. Users provide their own.

\begin{itemize}
    \item \textbf{Embedding Models:} User's choice (OpenAI, Hugging Face, Qwen, etc.)
    \item \textbf{LLM Providers:} OpenAI, Anthropic, LiteLLM gateway, custom servers
    \item \textbf{Key Management:} Users manage keys in browser (secure localStorage)
    \item \textbf{Backend Role:} Coordinates search; does \textit{not} call LLMs
    \item \textbf{Cost Model:} Users pay their provider directly
    \item \textbf{Team Testing:} Internal Qwen embedding model on dev server (free)
\end{itemize}

\textit{See Appendix B: LLM Strategy for full architecture details.}

\subsection{Storage \& External Services}
\begin{itemize}
    \item \textbf{File Storage:} AWS S3 (PDFs, Markdown, exports)
    \item \textbf{Email:} AWS SES (verification, password reset)
    \item \textbf{Caching:} AWS ElastiCache (future, for job queues)
    \item \textbf{Deployment:} AWS Amplify (auto-deploy from \texttt{prod} branch)
    \item \textbf{PDF Processing:} pdf-parse (text extraction)
    \item \textbf{OCR:} TBD (Google Vision or AWS Textract, deferred to Phase 2)
\end{itemize}

\subsection{Local Development}
\begin{itemize}
    \item \textbf{Docker:} \texttt{docker-compose up} (PostgreSQL + app)
    \item \textbf{Database:} PostgreSQL container (pgvector pre-installed)
    \item \textbf{File Storage:} AWS S3 (configure credentials in \texttt{.env.local})
    \item \textbf{Email:} Nodemailer (local SMTP or AWS SES test)
\end{itemize}

\section{Changes from Original Plan}

We changed a few things from the original spec. This isn't a failure---just responding to reality and team strengths.

\subsection*{Database: MariaDB → PostgreSQL + pgvector}
\textbf{Reason:} Team knows PostgreSQL. Both work for vector search, so no real loss.

\subsection*{Architecture: Microservices → Single Next.js App}
\textbf{Reason:} Faster to build, easier to deploy. We don't need the complexity for a semester project.

\subsection*{Background Jobs: Redis → Add Later}
\textbf{Reason:} Long-running PDF processing can be synchronous for now. Add async in Phase 2 if needed.

\subsection*{LLM Keys: Backend-Managed → User-Managed}
\textbf{Reason:} Users provide their own API keys. We never see them, they pay directly. Simpler for us, more control for them.

\section{How It's Built}

\subsection{Database}

PostgreSQL holds: users, notes, folders, PDFs. AWS S3 holds: uploaded files.

\textbf{Key tables:} users, notes, folders, attachments, annotations, (later) quizzes, flashcards.

\textbf{Search optimization:} Indexes on text content and vector embeddings for fast lookups.

\subsection{API Endpoints}

\textbf{Auth:} \texttt{/api/auth/register}, \texttt{/login}, \texttt{/logout}, \texttt{/forgot-password}

\textbf{Notes:} \texttt{GET /api/notes}, \texttt{POST /api/notes}, \texttt{GET/PATCH/DELETE /api/notes/:id}

\textbf{Files:} \texttt{POST /api/upload}, \texttt{GET /api/tree?sort=...}

\textbf{Search (Phase 1):} \texttt{GET /api/search?q=...}, \texttt{POST /api/notes/:id/embed}

\textbf{RAG (Phase 2):} \texttt{POST /api/rag/chat}, \texttt{GET /api/rag/search}

\textit{Details in SEARCH\_ARCHITECTURE\_PLAN.md}

\subsection{Services Used}

AWS S3 (files), AWS SES (email), AWS Amplify (deploy), AWS RDS (database), LLMs (your own keys).

\section*{Terms}

\textbf{RAG:} Ask questions, get answers from your notes.
\textbf{Embedding:} Vector (number list) representing text meaning.
\textbf{Chunking:} Break large PDFs into smaller pieces.
\textbf{Hybrid Search:} Keyword + meaning together.
\textbf{SM-2:} Flashcard scheduling algorithm.
\textbf{PWA:} Web app you can install.
\textbf{JWT:} Token proving you're logged in.
\textbf{MVP:} Ship by semester end.
