\section{What This Is}
%==============================================================================

OghmaNotes is a web-based learning platform for students to create Markdown notes, upload PDFs, search intelligently, and ask AI questions with answers backed by their own materials.

\textit{For:} Development team, supervisors, maintainers.

%==============================================================================
\section{Scope}
%==============================================================================

\subsection{MVP (Must Have)}

\begin{itemize}
    \item Web-based Markdown editor with offline support (PWA)
    \item PDF upload, annotation, text extraction
    \item Multi-mode search (fuzzy keyword + semantic/vector)
    \item RAG chat: ask questions, get answers with citations
    \item User authentication (email/password)
    \item Note organization (folder hierarchy)
    \item Responsive design (desktop, tablet)
\end{itemize}

\subsection{Out of Scope}

\begin{itemize}
    \item Light mode (dark only)
    \item Real-time collaborative editing
    \item Native mobile apps (PWA sufficient)
    \item Instructor dashboards
    \item Video/audio transcription
\end{itemize}

\subsection{Goals}

\begin{enumerate}
    \item Deliver MVP in 6 weeks
    \item Enable 3 people to work in parallel
    \item Auto-deploy from production branch
    \item Stay within AWS free tier + modest budget
\end{enumerate}

\section{Why We Changed Our Tech Stack}
%==============================================================================

The original plan specified MariaDB and microservices. We adapted based on team expertise and MVP pragmatism.

\subsection*{Database: MariaDB $\rightarrow$ PostgreSQL + pgvector}

PostgreSQL with pgvector extension. Team expertise and well-maintained support.

\subsection*{Architecture: Microservices $\rightarrow$ Next.js Monolith}

Single Next.js app (frontend + API routes). Faster iteration for MVP.

\subsection*{Background Jobs: Redis + BullMQ $\rightarrow$ Deferred}

Synchronous processing initially, async added later if needed.

\subsection*{LLM Key Management: Backend-Managed $\rightarrow$ User-Managed}

Users provide own keys. Simpler, transparent costs, supports any LLM.

\section{Database Schema}
%==============================================================================

\subsection{ID Strategy}

All ID columns use UUID v7 (Universally Unique Identifier version 7):
\begin{itemize}
    \item Format: \texttt{xxxxxxxx-xxxx-7xxx-yxxx-xxxxxxxxxxxx}
    \item Cryptographically secure (no collision risk)
    \item Sortable by timestamp
    \item Generated server-side
\end{itemize}

Affected columns:
\begin{itemize}
    \item \texttt{app.login.user\_id} --- User identifier
    \item \texttt{app.notes.note\_id} --- Note identifier
    \item \texttt{app.documents.id} --- Document identifier
    \item \texttt{app.chunks.id} --- Chunk identifier
\end{itemize}

\subsection{Core Tables}

\begin{itemize}
    \item \texttt{app.login} --- User accounts, hashed passwords
    \item \texttt{app.notes} --- Note content, metadata, search/embedding columns
    \item \texttt{app.tree\_items} --- Folder hierarchy
    \item \texttt{app.attachments} --- Uploaded PDFs and files
    \item \texttt{app.pdf\_annotations} --- Highlights, notes on PDFs
    \item \texttt{app.quiz\_questions} --- Quiz questions
    \item \texttt{app.flashcards} --- Flashcard decks
\end{itemize}

\subsection{Search Columns}

\texttt{app.notes} includes:
\begin{itemize}
    \item \texttt{extracted\_text} --- Full text from PDFs
    \item \texttt{embedding} --- Vector (1536-dim) for semantic search
    \item \texttt{search\_vector} --- PostgreSQL tsvector for full-text search
    \item \texttt{relative\_path} --- For organizing search results
\end{itemize}

\subsection{Indexes}

\begin{itemize}
    \item GIN index on \texttt{search\_vector} (full-text)
    \item ivfflat index on \texttt{embedding} (semantic)
    \item Compound indexes on \texttt{user\_id + created\_at} and \texttt{user\_id + updated\_at}
\end{itemize}

\section{API Endpoints}
%==============================================================================

\subsection{Authentication}

\begin{itemize}
    \item \texttt{POST /api/auth/register}
    \item \texttt{POST /api/auth/login}
    \item \texttt{POST /api/auth/logout}
    \item \texttt{POST /api/auth/forgot-password}
    \item \texttt{POST /api/auth/reset-password}
\end{itemize}

\subsection{Notes \& Files}

\begin{itemize}
    \item \texttt{GET /api/notes} --- List user's notes
    \item \texttt{POST /api/notes} --- Create note
    \item \texttt{GET /api/notes/:id} --- Get note
    \item \texttt{PATCH /api/notes/:id} --- Update note
    \item \texttt{DELETE /api/notes/:id} --- Delete note
    \item \texttt{POST /api/upload} --- Upload PDF/file
    \item \texttt{GET /api/tree} --- Get folder hierarchy
\end{itemize}

\subsection{Search}

\begin{itemize}
    \item \texttt{GET /api/search?q=QUERY\&type=fuzzy|semantic}
    \item \texttt{POST /api/notes/:id/embed} --- Generate embedding
\end{itemize}

\subsection{RAG}

\begin{itemize}
    \item \texttt{POST /api/rag/chat} --- Submit question, get answer
    \item \texttt{GET /api/rag/search} --- Semantic search for context
\end{itemize}
