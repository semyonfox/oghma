OghmaNotes Design System

Token-based design reference for maintaining visual consistency across the landing page and /notes app. Use this document when patching or extending any UI.

Design Philosophy

Dark, Focused Environment
The entire app is dark-mode only. There is no light mode. Never add dark: Tailwind prefixes — they are dead code. The background is #111827 (gray-900), matching the landing page exactly.
Semantic Tokens Always
Never hardcode Tailwind color scales (bg-gray-900, text-slate-400, etc.) in component files. Use the semantic token classes defined in tailwind.config.js — e.g., bg-background, text-text-secondary, border-border.
Indigo Primary Accent
All interactive highlights, active states, and CTAs use the indigo scale (primary-*). This matches the landing page's indigo-500/600. Never use blue-*, sky-*, or teal-* as accent colors.
Minimal, Study-Focused
Avoid decorative excess in the notes app. No gradient blobs (those are landing-page only). Borders should be subtle. Hover states use low-opacity white overlays, not color shifts.
Consistent Gray Harmony
The landing page uses Tailwind's gray-* scale. The semantic surface tokens map to gray-900/800/700. Avoid mixing slate-*, neutral-*, and gray-* across components.
No AI Templates
Avoid glassmorphism cards, animated gradients, glowing borders, or multi-color accent palettes. The aesthetic is clean, typographic, and functional — inspired by Tailwind UI dark templates, not generative dashboards.

Color Tokens

All tokens are defined in globals.css under @theme and mapped to Tailwind utility classes in tailwind.config.js.

Border Strategy

Two border tiers keep the layout visually clean without harsh lines:

TokenValueWhere to Use
border-border rgba(255,255,255,0.10) Panel-to-panel separators, modal edges, input focus rings, section dividers
border-border-subtle rgba(255,255,255,0.06) Internal item separators (tree items, list rows), header/content dividers within a panel

Never use hardcoded border-white/5, border-white/6, border-white/8, border-white/10, border-gray-700, or border-slate-700.

Patch Guide — How to Fix Future Inconsistencies

1. Check the token table first
Before using any color class, check section 5 (Token Migration Reference). If you're about to write bg-gray-*, text-slate-*, or any raw Tailwind gray — use the semantic token instead.
2. No dark: prefixes
The entire app is dark. If you see dark:bg-* or dark:text-* anywhere, remove the dark: prefix and keep just the base class (updating it to the semantic token).
3. New accent colors
If you need a new semantic color, add it to globals.css under @theme and register it in tailwind.config.js. Never use raw Tailwind color names as accent in components.
4. Updating the primary color
To change the accent from indigo to something else, update only the --color-primary-* values in globals.css. All components using primary-* classes will update automatically.
5. New components
Start from the patterns in section 6. Use bg-surface for the container, border-border-subtle for internal dividers, text-text-secondary for default text, and text-text-tertiary for muted labels.
6. Landing vs. Notes
The landing page (page.js, header.jsx, etc.) now uses semantic tokens too. The gradient blobs (from-[#ff80b5] to-[#9089fc]) are intentionally landing-page-only decorations — do not copy them into notes components.
