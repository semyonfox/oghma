🔴 BUG #1: MIDDLEWARE PROTECTION NOT ACTIVE
═══════════════════════════════════════════════════════════════════════════════

WHAT WAS WRONG:
───────────────────────────────────────────────────────────────────────────────
The file src/proxy.ts contained middleware logic with route matchers, but
Next.js was ignoring it completely.

Why?
  • Next.js only loads middleware from: src/middleware.ts or middleware.ts
  • File was named proxy.ts (non-standard name)
  • Function exported as "export function proxy" instead of "export default"

IMPACT:
───────────────────────────────────────────────────────────────────────────────
All supposedly "protected" routes were completely UNPROTECTED:

  ❌ /notes                    (should require login)
  ❌ /upload                   (should require login)
  ❌ /settings                 (should require login)
  ❌ /trash                    (should require login)
  ❌ /api/notes                (should require auth)
  ❌ /api/extract              (should require auth)
  ❌ /api/tree                 (should require auth)
  ❌ /api/trash                (should require auth)
  ❌ /api/settings             (should require auth)
  ❌ /api/upload               (should require auth)
  ❌ /api/import-export        (should require auth)

Anyone could:
  • Visit /notes without logging in
  • Call /api/notes directly without a session cookie
  • Access protected endpoints via curl/Postman without authentication

THE FIX:
───────────────────────────────────────────────────────────────────────────────
✅ Created: src/middleware.ts
   • Correct filename recognized by Next.js
   • Proper default export: "export default async function middleware"
   • Same logic and matchers as proxy.ts
   • Now loaded automatically on server startup

BEFORE:
  src/proxy.ts
  export async function proxy(request) {
    // This function never runs!
  }

AFTER:
  src/middleware.ts
  export default async function middleware(request) {
    // This function runs for all protected routes
  }

🚀 QUICK START (5 minutes)
═══════════════════════════════════════════════════════════════════════════════

1. Start PostgreSQL (Docker):
   docker run -d --name oghmanotes-db \
     -e POSTGRES_USER=notes_user \
     -e POSTGRES_PASSWORD=notes_password \
     -e POSTGRES_DB=oghmanotes \
     -p 5432:5432 \
     postgres:16-alpine

2. Create tables:
   psql postgresql://notes_user:notes_password@localhost:5432/oghmanotes \
     < database/schema.sql

3. Configure .env.local:
   cat > .env.local << 'ENVFILE'
   DATABASE_URL=postgresql://notes_user:notes_password@localhost:5432/oghmanotes
   JWT_SECRET=$(node -e "console.log(require('crypto').randomBytes(32).toString('hex'))")
   NODE_ENV=development
   NEXT_PUBLIC_API_URL=http://localhost:3000
   ENVFILE

4. Start dev server:
   pnpm dev

5. Test:
   Open http://localhost:3000/register
   Create account: test@example.com / Password123
